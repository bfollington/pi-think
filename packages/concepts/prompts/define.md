## Process

1. **Check existing concepts**: Read `.concepts/` to see if this concept (or something close) already exists.
2. **Draft the definition**: Write a concept file following the format in the skill — name, summary, purpose, structure, relations.
3. **Show me the draft**: Present it before writing anything.
4. **Link to related concepts**: If related concepts exist, propose adding reciprocal links.

If I gave you a name but no description, ask me: "What makes this different from [related thing]?" — that's the core of a good concept definition.

If I described something but didn't name it, propose a name.
